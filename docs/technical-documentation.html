Technology Decision Rationale

Why React Native + Expo?

We chose React Native with Expo for its rapid development cycle. The managed workflow eliminates native configuration complexity, while EAS Build provides professional-grade CI/CD.

Why Dual Gemini Models?

We use **Gemini 2.0 Flash** for text tasks (stable, fast, cost-effective) and **Gemini 3 Flash Preview** for vision/multimodal (advanced image understanding). This separation optimizes both performance and cost. Multi-key failover ensures 99.9% uptime.

Why Zustand?

Zustand provides a simpler mental model than Redux. For SousChef's use cases (shopping lists, timers), its hooks-based API integrates seamlessly with React Native with minimal bundle size impact.

Code Logic: Entitlement Check

Before allowing AI recipe import, we validate the user's subscription status:

// In URLImportModal.tsx
const handleImportRecipe = async (url: string) => {
  try {
    const importCount = await getImportCount();
    const { customerInfo } = await Purchases.getCustomerInfo();
    
    // Check for active pro_access entitlement
    const isPro = customerInfo.entitlements.active['pro_access'] !== undefined;
    const FREE_LIMIT = 5;

    if (!isPro && importCount >= FREE_LIMIT) {
      setShowPaywall(true);
      return;
    }

    await importRecipeFromURL(url);
    await incrementImportCount();
  } catch (error) {
    console.error('Import failed:', error);
  }
};

AI Workflow & Model Architecture

SousChef employs a dual-model AI architecture: Gemini 2.0 Flash for text-based tasks (meal planning, substitutions, cooking tips) and Gemini 3 Flash Preview for vision/multimodal tasks (video/image recipe extraction, portion analysis).

Model Selection Strategy

Gemini 2.0 Flash (Text Model)

Use Cases: Recipe text extraction, cooking tips, ingredient substitutions, meal plan generation, dietary advice.

Benefits: Stable, fast, cost-effective. Optimized for pure-text tasks.

Gemini 3 Flash Preview (Vision Model)

Use Cases: Video recipe extraction, image-based recipe analysis, portion analysis, content bundle analysis with media.

Benefits: Advanced multimodal understanding. 1M+ token context window.

Multi-Key Failover System

SousChef implements automatic API key rotation with up to 5 keys. When one key reaches quota (429), hits rate limits, or returns auth errors (401/403), the system automatically rotates to the next available key. Ensures 99.9% uptime for AI features.

Video-to-Recipe Extraction

Converting unstructured social media video content (TikTok, Instagram Reels) into structured recipe data using Gemini 3 Flash Preview's multimodal capabilities.

1. Video Download & Caching

Download temporary cached version respecting platform TOS. Files are cached locally in FileSystem.cacheDirectory.

2. Multimodal Extraction

Extract frames at 0%, 25%, 50%, 75%, 100% and transcribe audio using expo-av.

3. Gemini Vision Processing

Send visual frames + audio transcript to Gemini 3 Flash Preview (vision model) with a strict JSON schema prompt. Model analyzes visual cues and audio simultaneously for complete recipe extraction.

4. Validation

Validate the output against our Recipe schema using Zod to ensure type safety before saving.

Security & Privacy

Privacy Principles

AI processing is stateless. We do not store raw user videos on servers—only the extracted metadata. Temporary caches are cleared every 24 hours.

Implementation Details

  • Secure Storage: API keys and tokens stored in Expo SecureStore.
